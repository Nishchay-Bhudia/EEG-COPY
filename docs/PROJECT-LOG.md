EEG-UI (Express + static HTML/JS, single-user Muse EEG monitor, persists to Supabase Postgres via `api/server.js`) is being evolved to be functionally like the `~/projects/eeg-controlhub` mockup (a teacher "control hub" — 7 sidebar views, rich SVG instruments, a prescribe wizard), wired to the `~/projects/EEG-BACKEND` .NET 8 classifier.

**Key architectural decisions:** (1) Express/Postgres stays the single source of truth; .NET stays a stateless classifier — do NOT dual-write epochs. (2) Model the cohort in Express, not .NET — a `clients` table + nullable `eeg_sessions.client_id`. (3) One `showView()` router replaces the old `show*Screen` toggles.

**Phased roadmap:** P0 shell+router+token merge · P1 cohort/client tables+endpoints · P2 home/cohort/client views · P3 analyze SVG instruments · P4 prescribe wizard+AI · P5 replay view · P6 (optional) SignalR live streaming.

**Headband driver layer (2026-07-10):** Refactored `app.js` BT acquisition into a driver/adapter pattern so multiple headbands are supported with a device-agnostic consumption pipeline. `DRIVERS` registry; each driver declares `{id,name,sampleRate,channelCount,filters,optionalServices,isMatch(server),start(server,ctx)}` and funnels decoded µV through the generic `pushSamples(ch, µV[])` sink → unchanged `processBluetoothEEG`/analyze/render/store. `connectBluetooth()` now scans the union of driver filters, matches a driver by primary service, sizes `bleChannels` to `channelCount`, sets mutable `activeSampleRate` (used in the `/analyze` POST body + local `bandPowers` fallback). Both `MuseDriver` and `BrainBitDriver` are registered. **BrainBit driver DONE (2026-07-10):** protocol reverse-sourced from the npm `brainbit-client`/`web-neurosdk-brainbit` web SDK (not from unreliable web guides — the "reverse-engineering-ble-devices" readthedocs turned out to be about a radiator valve, not EEG). Real protocol: service `6e400001-b534-f393-68a9-e50e24dcca9e`, command char `…0003` (write), signal/EEG char `…0004` (notify); start = write `[2,0,0,0,0]` (we send stop `[1]` first); 20-byte packets = 4 ch × 2 samples, sample0 is a 20-bit bit-packed value left-aligned into a signed int32 then /2048, sample1 is a 16-bit signed delta added to sample0; volts scale `2.4/(0xFFFFF*6)`. Scanned by `namePrefix:'BrainBit'` (doesn't advertise its service). Our inline decoder was verified **byte-exact** vs the reference `parseSignalValuesShifts` across 6 edge-case packets (scratchpad `verify_bb.js`). NOT yet tested against a physical band in-browser (Web Bluetooth needs Windows Chrome/Edge; user has the device). BrainBit resistance/electrode-contact mode exists in the SDK but is not wired up (EEG signal only).
